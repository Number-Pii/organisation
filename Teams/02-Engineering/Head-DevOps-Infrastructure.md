# Head of DevOps & Infrastructure

## Position Details
- **Department:** Engineering
- **Reports To:** CTO
- **Direct Reports:** Senior DevOps Engineer(s), Senior Cloud Engineer
- **Employment Type:** Full-time

## Role Summary
Owns all infrastructure, deployment pipelines, and platform reliability. Has designed multi-region, high-availability infrastructure at scale. Ensures every product and client project ships on robust, secure, and scalable infrastructure.

## Core Skills
- Infrastructure architecture and design
- CI/CD pipeline design and optimisation
- Cloud cost management and optimisation
- High availability and disaster recovery
- Container orchestration
- Monitoring, alerting, and incident response
- Infrastructure as Code

## Technical Skills
- AWS (EC2, ECS, Lambda, RDS, S3, CloudFront, Route 53, VPC)
- Docker, Kubernetes, ECS Fargate
- Terraform, CloudFormation
- GitHub Actions, GitLab CI, Jenkins
- Vercel, Netlify (static deployments)
- Datadog, CloudWatch, PagerDuty
- Nginx, load balancers, CDN configuration
- Linux server administration

## Project Involvement
| Project Type | Role in Project | Authority Level |
|---|---|---|
| Infrastructure Scaling | Lead | Approve |
| New Product Deployment | Infrastructure setup | Execute |
| Client Project Deployment | Infrastructure reviewer | Advise |
| Incident Response | On-call lead | Execute |
| Cost Optimisation | Owner | Approve |

## Approval Authority
- **Can approve:** Infrastructure changes, deployment pipeline modifications, cloud resource provisioning, DevOps hires
- **Needs approval from:** CTO (major architecture changes, large cost commitments)
