# Head of Information Security

## Position Details
- **Department:** Engineering
- **Reports To:** CTO
- **Direct Reports:** Senior Security Engineer, Security Analyst
- **Employment Type:** Full-time

## Role Summary
Owns the security posture of all Number Pii products, infrastructure, and client deliveries. Designs and enforces security policies, conducts security reviews, manages vulnerability response, and ensures compliance with industry standards. This role is critical for protecting Thirty X user data and maintaining client trust.

## Core Skills
- Security architecture and threat modelling
- Vulnerability management and incident response
- Security policy development and enforcement
- Compliance frameworks (GDPR, SOC 2, ISO 27001)
- Risk assessment and mitigation
- Security awareness training
- Penetration testing programme management
- Vendor security assessment

## Technical Skills
- OWASP Top 10 and application security
- SAST/DAST tools (Snyk, SonarQube, Checkmarx, Burp Suite)
- Cloud security (AWS Security Hub, IAM, GuardDuty, Config)
- Network security (firewalls, WAF, DDoS protection)
- Authentication & authorisation (OAuth 2.0, OIDC, SAML, JWT)
- Container security (Docker, Kubernetes security policies)
- SIEM and log analysis
- Encryption (TLS, at-rest, key management — KMS)
- Content Security Policy, CORS, HSTS
- Incident response playbooks

## Project Involvement
| Project Type | Role in Project | Authority Level |
|---|---|---|
| Security Audits / Pen Tests | Lead | Approve |
| Thirty X Security Reviews | Owner | Approve (security sign-off) |
| Client Project Security | Reviewer | Advise / Approve |
| Incident Response | Lead | Execute |
| Compliance Initiatives | Owner | Approve |
| Infrastructure Security | Reviewer | Advise |
| New Product Security Assessment | Lead | Approve |

## Approval Authority
- **Can approve:** Security policies, vulnerability response actions, compliance standards, security tool selection, security hires
- **Needs approval from:** CTO (security budget, major architectural security changes), CEO (compliance programme commitments)
