# Head of QA & Reliability

## Position Details
- **Department:** Engineering
- **Reports To:** CTO
- **Direct Reports:** Senior QA Engineer(s), QA Automation Engineer
- **Employment Type:** Full-time

## Role Summary
Builds bulletproof test infrastructure and owns the quality bar across all products and client deliveries. Not just a test runner — this role designs the quality strategy, automated testing frameworks, and release gates that ensure nothing ships broken.

## Core Skills
- Test strategy and planning (@testing-qa, @test-driven-development)
- Automated testing framework design (@e2e-testing, @javascript-testing-patterns)
- Quality metrics and reporting (@analytics-product, @grafana-dashboards)
- Release management and quality gates (@cicd-automation-workflow-automate, @deployment-validation-config-validate)
- Performance and load testing (@k6-load-testing, @performance-testing-review-ai-review)
- Bug triage and severity assessment (@testing-qa, @systematic-debugging)
- Cross-team quality advocacy (@internal-comms, @team-collaboration-standup-notes)

## Technical Skills
- Playwright, Cypress (E2E testing) (@playwright-skill, @e2e-testing)
- Jest, Vitest (unit testing) (@javascript-testing-patterns, @tdd-workflows-tdd-cycle)
- Testing Library (component testing) (@javascript-testing-patterns)
- k6, Artillery (load testing) (@k6-load-testing)
- Lighthouse, WebPageTest (performance testing) (@web-performance-optimization)
- CI/CD test pipeline integration (@cicd-automation-workflow-automate, @github-actions-templates)
- API testing (Postman, REST Client) (@api-design-principles)
- Mobile testing (Appium, Detox) (@e2e-testing)

- **Can approve:** Release readiness, test strategy, QA hires, quality standards
- **Needs approval from:** CTO (release decisions with known issues), VP Product/Client Eng (scope changes based on quality findings)
