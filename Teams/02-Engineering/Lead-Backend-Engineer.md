# Lead Backend Engineer

## Position Details
- **Department:** Engineering — Product Engineering
- **Reports To:** VP of Product Engineering
- **Direct Reports:** Senior Backend Engineer(s)
- **Employment Type:** Full-time

## Role Summary
The technical lead for all backend engineering on Number Pii products. Owns API design, database architecture, data models, and backend performance for Thirty X and future products.

## Core Skills
- Backend architecture and API design
- Database design and optimisation
- Authentication and authorisation systems
- Microservices and event-driven architecture
- Code review and mentorship
- Data migration and versioning strategies
- Performance profiling and caching strategies

## Technical Skills
- Node.js, Express, Fastify, NestJS
- PostgreSQL (schema design, query optimisation, migrations)
- GraphQL, REST API design
- Redis (caching, sessions, queues)
- OAuth 2.0, JWT, OIDC (authentication)
- Prisma, Drizzle ORM, Knex
- Bull/BullMQ (job queues)
- AWS (RDS, Lambda, SQS, SNS, S3)
- Jest, Supertest (API testing)
- Docker (containerised services)

## Project Involvement
| Project Type | Role in Project | Authority Level |
|---|---|---|
| Thirty X Backend | Technical lead | Approve (backend decisions) |
| API Design | Owner | Approve |
| Database Schema | Owner | Approve |
| Backend Code Reviews | Reviewer | Approve |
| Third-Party Integrations | Lead | Execute |

## Approval Authority
- **Can approve:** Backend architectural decisions, API contracts, database migrations, PR merges
- **Needs approval from:** VP Product Engineering (major schema changes), Head of InfoSec (security-sensitive changes)
