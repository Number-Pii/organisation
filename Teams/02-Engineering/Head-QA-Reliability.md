# Head of QA & Reliability

## Position Details
- **Department:** Engineering
- **Reports To:** CTO
- **Direct Reports:** Senior QA Engineer(s), QA Automation Engineer
- **Employment Type:** Full-time

## Role Summary
Builds bulletproof test infrastructure and owns the quality bar across all products and client deliveries. Not just a test runner — this role designs the quality strategy, automated testing frameworks, and release gates that ensure nothing ships broken.

## Core Skills
- Test strategy and planning
- Automated testing framework design
- Quality metrics and reporting
- Release management and quality gates
- Performance and load testing
- Bug triage and severity assessment
- Cross-team quality advocacy

## Technical Skills
- Playwright, Cypress (E2E testing)
- Jest, Vitest (unit testing)
- Testing Library (component testing)
- k6, Artillery (load testing)
- Lighthouse, WebPageTest (performance testing)
- CI/CD test pipeline integration
- API testing (Postman, REST Client)
- Mobile testing (Appium, Detox)

## Project Involvement
| Project Type | Role in Project | Authority Level |
|---|---|---|
| Thirty X Releases | Quality gatekeeper | Approve (release readiness) |
| Client Project Delivery | Quality reviewer | Approve (delivery quality) |
| Test Infrastructure | Owner | Approve |
| CI/CD Quality Gates | Owner | Approve |
| Performance Benchmarks | Owner | Execute |

## Approval Authority
- **Can approve:** Release readiness, test strategy, QA hires, quality standards
- **Needs approval from:** CTO (release decisions with known issues), VP Product/Client Eng (scope changes based on quality findings)
